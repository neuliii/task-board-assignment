# AI 도구 활용 (AI_USAGE)

> AI 사용은 **권장**됩니다. 이 문서는 "얼마나 썼는지"가 아니라
> **AI가 준 답을 어떻게 검증·판단했는지**를 봅니다. 솔직하게 적어 주세요.

## 1. 사용한 도구

* OpenAI Codex
* ChatGPT (GPT-5.5)


## 2. 주로 어디에 썼나

| 작업 | 사용한 도구 | 활용 방식 |
|------|-------------|-----------|
| 낙관적 업데이트 및 Rollback 구현 | OpenAI Codex | 구현 초안 작성 및 코드 생성 |
| CRUD 기능 구현 | OpenAI Codex | 기존 구조를 기반으로 기능 구현 |
| 리스트 가상화 적용 | OpenAI Codex | 라이브러리 적용 및 구현 보조 |
| 설계 검토 및 구현 방향 확인 | ChatGPT | 구현 전 요구사항 해석 및 설계 검토 |
| 테스트 코드 작성 | OpenAI Codex | 테스트 코드 초안 작성 후 직접 수정 |


## 3. AI 답을 그대로 쓰지 않고 검증·거부·수정한 사례

### 사례 1. 불필요한 구조 변경 제안
AI는 일부 기능 구현 과정에서 기존 구조를 변경하거나 새로운 상태 관리 방식을 추가하는 방향을 제안했습니다.
하지만 과제의 목적은 기존 프로젝트를 확장하는 것이었기 때문에 기존 Zustand Store 구조를 유지하도록 직접 수정했습니다.

### 사례 2. 불필요한 리팩토링 제거
AI는 기능 구현과 함께 여러 컴포넌트를 분리하거나 추상화하는 리팩토링을 제안했습니다.
과제에서는 기능 구현보다 기존 구조를 유지하는 것이 더 적절하다고 판단하여 필요한 코드만 적용하고 나머지는 사용하지 않았습니다.

### 사례 3. 과제 요구사항 검증
AI가 작성한 코드가 과제 요구사항(Priority 1)을 모두 만족하는지 직접 확인했습니다.
특히 다음 항목을 중점적으로 검증했습니다.
* 실패 시 정확한 Rollback이 수행되는지
* Race Condition이 발생해도 최신 요청만 반영되는지
* TypeScript(strict) 환경에서 타입 오류가 없는지
* 기존 프로젝트 구조를 유지하는지

검증 과정에서 필요한 부분은 직접 수정하여 적용했습니다.


## 4. 스스로 판단해서 결정한 부분

다음 항목은 AI 제안과 별개로 과제 요구사항과 프로젝트 규모를 고려하여 직접 결정했습니다.

* 상태 관리는 Redux 대신 Zustand를 사용했습니다.
* 대량 데이터 렌더링은 @tanstack/react-virtual을 사용했습니다.
* Priority 2 기능보다 Priority 1의 완성도를 높이는 방향으로 구현 범위를 결정했습니다.
* Rollback은 서버 재조회 대신 이전 상태 스냅샷을 복원하는 방식을 선택했습니다.
* 자동 재시도나 추가적인 추상화는 구현하지 않고 기존 구조를 최대한 유지했습니다.

AI는 구현을 보조하는 도구로 활용했으며, 최종 설계와 구현 방식은 과제 요구사항과 코드의 일관성을 기준으로 직접 판단하여 결정했습니다.
